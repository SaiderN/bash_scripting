## Case-инструкция (альтернатива if-elif)
Инструкция `case` позволяет сравнивать одну переменную с множеством значений. Это более чистая альтернатива длинным цепочкам `if-elif-elif`.
**Структура:**
```bash
case "$переменная" in
    шаблон1)
        # команды если переменная соответствует шаблон1
        ;;
    шаблон2)
        # команды если переменная соответствует шаблон2
        ;;
    шаблон3|шаблон4)  # несколько шаблонов через |
        # команды если переменная соответствует шаблон3 ИЛИ шаблон4
        ;;
    *)
        # команды по умолчанию (если ни один шаблон не подошёл)
        ;;
esac
```
**Не слушайте гениев, которые говорят что можно писать каждую команду через `;;`. Можно только в конце каждого шаблона 1 раз. Хотя вам стоит самим увидеть эту синтаксическую ошибку. Попробуйте вариант из моего примера и найдите как можно больше вариантов синтаксических ошибок. Так получится запомнить**
Пример:
```bash
#!/bin/bash
read -p "Выберите действие (start|stop|restart|status): " action

case "$action" in
    start)
        echo "Запуск службы..."
        # команды запуска
        ;;
    stop)
        echo "Остановка службы..."
        # команды остановки
        ;;
    restart)
        echo "Перезапуск службы..."
        # команды перезапуска
        ;;
    status)
        echo "Проверка статуса службы..."
        # команды проверки
        ;;
    *)
        echo "Неизвестное действие: $action"
        echo "Доступные действия: start, stop, restart, status"
        exit 1
        ;;
esac
```
## Case vs If-elif
**Когда использовать case:**

Сравниваете одну переменную с несколькими фиксированными значениями

Работаете с шаблонами (`wildcards`) типа `*.txt, backup_*`

Нужна более чистая и читаемая структура

**Когда использовать `if-elif`:**

Нужны сложные условия (сравнение нескольких переменных)

Требуются арифметические операции в условиях

Нужны проверки файлов (`-f, -d`, и т.д.)

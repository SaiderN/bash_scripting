## Условные операторы (if, elif, else)

### Базовый синтаксис
Условный оператор `if` позволяет выполнять различные команды в зависимости от условий.

**Структура:**
```bash
if [[ условие1 ]]; then
    # команды если условие1 истинно
elif [[ условие2 ]]; then
    # команды если условие2 истинно
else
    # команды если все условия ложны
fi
```

### Варианты условий (тестов)
**1. Числовые сравнения (только для целых чисел):**
```bash
# -eq равно (equal)
[[ $a -eq $b ]]  # a == b

# -ne не равно (not equal)  
[[ $a -ne $b ]]  # a != b

# -lt меньше (less than)
[[ $a -lt $b ]]  # a < b

# -le меньше или равно (less or equal)
[[ $a -le $b ]]  # a <= b

# -gt больше (greater than)
[[ $a -gt $b ]]  # a > b

# -ge больше или равно (greater or equal)
[[ $a -ge $b ]]  # a >= b
```
**2. Строковые сравнения:**
```bash
# = равно (сравнение строк)
[[ "$str1" = "$str2" ]]

# != не равно
[[ "$str1" != "$str2" ]]

# < меньше (в лексикографическом порядке)
[[ "$str1" < "$str2" ]]

# > больше (в лексикографическом порядке)  
[[ "$str1" > "$str2" ]]

# -z строка пуста (zero length)
[[ -z "$str" ]]  # истинно если строка пустая

# -n строка не пуста (non-zero length)
[[ -n "$str" ]]  # истинно если строка не пустая
```
**3. Проверки файлов и директорий:**
```bash
# -e файл существует (exists)
[[ -e "/path/to/file" ]]

# -f обычный файл (regular file)
[[ -f "/path/to/file" ]]

# -d директория (directory)
[[ -d "/path/to/dir" ]]

# -r файл доступен для чтения (readable)
[[ -r "/path/to/file" ]]

# -w файл доступен для записи (writable)
[[ -w "/path/to/file" ]]

# -x файл исполняемый (executable)
[[ -x "/path/to/file" ]]

# -s файл не пустой (size > 0)
[[ -s "/path/to/file" ]]

# -L символическая ссылка (symbolic link)
[[ -L "/path/to/link" ]]

# файл1 новее файла2 (newer than)
[[ "/path/to/file1" -nt "/path/to/file2" ]]

# файл1 старше файла2 (older than)
[[ "/path/to/file1" -ot "/path/to/file2" ]]
```
### Логические операторы
**Внутри двойных скобок [[ ]]:**
```bash
# И (AND) - оба условия должны быть истинны
[[ условие1 && условие2 ]]

# ИЛИ (OR) - хотя бы одно условие должно быть истинно  
[[ условие1 || условие2 ]]

# НЕ (NOT) - инвертирование условия
[[ ! условие ]]
```
**Пример с логическими операторами:**
```bash
#!/bin/bash
read -p "Введите ваш возраст: " age

if [[ $age -ge 0 && $age -lt 12 ]]; then
    echo "Вы ребёнок"
elif [[ $age -ge 12 && $age -lt 18 ]]; then
    echo "Вы подросток"
elif [[ $age -ge 18 && $age -lt 60 ]]; then
    echo "Вы взрослый"
elif [[ $age -ge 60 && $age -lt 120 ]]; then
    echo "Вы пожилой человек"
else
    echo "Возраст введён некорректно!"
fi
```
### Старый vs новый синтаксис
**Старый синтаксис (совместимый, но менее мощный):**
```bash
# Одинарные квадратные скобки [ ]
if [ "$a" -eq "$b" ]; then
    echo "a равно b"
fi

# Для строковых сравнений обязательно кавычки!
if [ -n "$str" ]; then
    echo "Строка не пустая"
fi
```
**Новый синтаксис (рекомендуется, Bash 2.0+):**
```bash
# Двойные квадратные скобки [[ ]]
if [[ $a -eq $b ]]; then
    echo "a равно b"
fi

# Не требует кавычек для переменных (но их лучше ставить)
if [[ -n $str ]]; then
    echo "Строка не пустая"
fi
```
Различия:

`[[ ]]` поддерживает `&&, ||` внутри без escape-символов

`[[ ]]` безопаснее с пустыми переменными

`[[ ]]` поддерживает регулярные выражения

`[ ]` более переносимый между разными оболочками
